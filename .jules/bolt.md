## 2025-03-16 - Avoid full state array dependencies in React useEffect for derived API requests
**Learning:** In Zustand or any global state manager, a component listening to a state array (`items`) that contains frequently updated properties (like `quantity`) will trigger unnecessary `useEffect` executions if the entire array is passed to the dependency array, even if the properties needed for the API call (like `items[0]?.category.slug`) haven't fundamentally changed. This leads to redundant API requests and UI sluggishness on simple cart quantity increments.
**Action:** Extract the primitive or strictly derived values (`const categorySlug = items[0]?.product.category?.slug`) during the render phase and use those primitives in the `useEffect` dependency array instead of the entire array. Handle local filtering operations against global state asynchronously or during render phase by using `useMemo`.
